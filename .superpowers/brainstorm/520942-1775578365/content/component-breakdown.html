Component Boundaries

Each box is a reusable piece

F

event_filters/1

Function component. Renders the filter bar (type dropdown, time range, text search, correlation fields). Emits phx-change events. Stateless — receives current filter values as assigns.

Embeddable anywhere a filter UI is needed. Host controls which filters are visible via an :enabled assign.

L

event_list/1

Function component. Renders the table of events with pagination controls. Receives @events, @total_count, @page. Each row links to /events/:id.

Embeddable in any page that has a list of events to display. Works with any filtered result set.

R

event_row/1

Function component. Single event row — type badge, timestamp, correlation indicator, one-line summary. Used inside event_list/1 but also independently (e.g., "recent events" widget).

The summary line is computed by an extracted pure function: EventsHelpers.event_summary/1

E

EventExplorer (LiveComponent)

The composition shell. Owns filter state, calls Events.list_events/1, manages pagination. Composes event_filters/1 + event_list/1. Host pages embed this with preset filters.

Embedding: <.live_component module={EventExplorer} id="match-events" preset={%{match_id: @match_id}} />
